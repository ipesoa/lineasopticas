LÍNEAS ÓPTICAS — CONTRATO DE PUBLICACIÓN, SEO Y GENERADOR SVG
===============================================================

OBJETIVO
--------
El JSON sigue siendo la única fuente de verdad. La IA envía datos estructurados
mediante el token de GitHub. Cada push a main activa GitHub Actions, valida el
JSON y genera automáticamente:

- una página HTML pública por noticia;
- una URL limpia /noticias/slug/;
- un SVG cuadrado ultraligero por noticia;
- sitemap.xml;
- news-sitemap.xml con noticias de las últimas 48 horas;
- feed.xml RSS;
- páginas de archivo y sección;
- metadatos canonical, Open Graph y NewsArticle.

La IA NO debe fabricar HTML, etiquetas meta ni código SVG dentro del artículo.
Eso lo genera scripts/build_site.py para evitar errores y duplicaciones.

CODIFICACIÓN Y TRANSPORTE
-------------------------
- UTF-8.
- Preferentemente JSON; también puede utilizarse JSONL.
- Token únicamente en la cabecera HTTP:

  Authorization: Bearer TU_TOKEN

- Nunca colocar el token en el JSON, en el artículo, en un SVG ni en un TXT.
- La respuesta automática debe contener solo datos válidos, sin Markdown.

CAMPOS PARA UNA NOTICIA NUEVA
-----------------------------
Campos obligatorios enviados por la IA:

- action: "add"
- title: titular editorial visible.
- seoTitle: variante explícita para buscadores.
- summary: resumen informativo de una o dos frases.
- metaDescription: descripción de buscador.
- content: artículo completo en texto plano.
- publishedAt: fecha ISO 8601 con zona horaria.
- updatedAt: inicialmente igual a publishedAt.
- author: firma visible.
- authorType: "Person" u "Organization".
- section: sección editorial principal.
- featured: true o false.
- tags: etiquetas temáticas visibles para clasificación interna.
- hiddenTags: sinónimos, entidades y búsquedas relacionadas.
- sources: lista de fuentes consultadas.
- language: "es".
- status: "published".

El sistema receptor debe crear id y slug estables al ejecutar action "add".
No deben cambiarse al corregir una noticia porque forman parte de su URL.

FORMATO RECOMENDADO
-------------------
{
  "action": "add",
  "title": "Dos días después del huracán",
  "seoTitle": "Qué ocurre dos días después del huracán",
  "summary": "Balance de daños, respuesta institucional y situación de las personas afectadas.",
  "metaDescription": "Daños, respuesta institucional y situación de la población dos días después del huracán.",
  "content": "Primer párrafo.\n\nSegundo párrafo.\n\nTercer párrafo.\n\nCuarto párrafo.",
  "publishedAt": "2026-07-19T20:30:00+02:00",
  "updatedAt": "2026-07-19T20:30:00+02:00",
  "author": "Líneas Ópticas",
  "authorType": "Organization",
  "section": "Actualidad",
  "featured": false,
  "tags": ["huracán", "emergencias", "clima"],
  "hiddenTags": ["daños", "evacuaciones", "respuesta institucional"],
  "sources": [
    {
      "name": "Nombre de la fuente",
      "url": "https://ejemplo.org/documento"
    }
  ],
  "language": "es",
  "status": "published"
}

REGLAS EDITORIALES Y DE POSICIONAMIENTO
---------------------------------------
TITLE
- Titular visible, editorial y legible.
- Preferencia: 3 a 10 palabras.
- Evitar titulares vagos que no indiquen el acontecimiento o la idea central.
- No copiar literalmente el titular de otra publicación.

SEOTITLE
- Entre 40 y 70 caracteres cuando sea posible.
- Debe contener la entidad, hecho o pregunta que una persona buscaría.
- Puede ser más explícito que title, pero no debe ser clickbait.
- No añadir "Líneas Ópticas": el compilador añade el nombre cuando corresponde.

SUMMARY
- Entre 110 y 220 caracteres.
- Debe explicar qué ha ocurrido y por qué importa.
- No repetir literalmente el primer párrafo.

METADESCRIPTION
- Aproximadamente 120 a 160 caracteres.
- Informativa, concreta y diferente entre noticias.
- Sin listas de palabras clave ni llamadas publicitarias.

CONTENT
- Texto plano; no enviar HTML.
- Separar párrafos mediante dos saltos de línea: \n\n.
- Abrir con información concreta; después aportar contexto, contradicción y el
  ángulo propio de Líneas Ópticas.
- Evitar reescribir una única fuente. Contrastar varias cuando el tema lo exija.
- No inventar cifras, citas, nombres, ubicaciones o fechas.
- Cuando un dato no esté confirmado, explicitar la incertidumbre.
- Una actualización sustancial debe modificar updatedAt y conservar id y slug.

SOURCES
- Cada elemento debe contener name y una URL pública HTTPS.
- Preferir fuentes primarias: documentos oficiales, declaraciones completas,
  datos originales, investigaciones o la publicación que originó la información.
- Las fuentes aparecen enlazadas en la página individual.

TAGS Y HIDDENTAGS
- tags: conceptos principales; no más de 8 normalmente.
- hiddenTags: nombres alternativos, siglas, lugares, personas y frases de búsqueda.
- No introducir etiquetas no relacionadas solo para atraer tráfico.

ACCIONES
--------
Nueva noticia:
{
  "action": "add",
  ...campos de la noticia...
}

Actualizar una noticia existente:
{
  "action": "update",
  "id": "identificador-existente",
  "updatedAt": "2026-07-19T22:15:00+02:00",
  ...solo campos que cambian o el registro completo, según acepte el receptor...
}

Eliminar:
{
  "action": "delete",
  "id": "identificador-existente"
}

No inventar id para update o delete.

VARIAS NOTICIAS
---------------
{
  "articles": [
    { "action": "add", "title": "...", "content": "..." },
    { "action": "add", "title": "...", "content": "..." }
  ]
}

JSONL
-----
Una noticia por línea. No usar corchetes ni comas entre líneas.

GENERADOR SVG — TITLE GRID V1
=============================

FUNCIÓN
-------
Crear una identidad cuadrada distinta para cada noticia usando únicamente el
titular. El SVG se muestra pequeño entre COMPARTIR y COPIAR LINK. Al pulsarlo,
abre la misma acción nativa de compartir; nunca descarga el archivo.

ENTRADA DEL PLUGIN
------------------
{
  "title": "Dos días después del huracán",
  "slug": "dos-dias-despues-del-huracan"
}

SALIDA DEL PLUGIN
-----------------
- Archivo: media/noticias/dos-dias-despues-del-huracan.svg
- MIME: image/svg+xml
- Dimensiones declaradas: 1200 x 1200.
- Fondo: #f7f7f5.
- Texto: #2d2d2b.
- Fuente CSS: Arial, Helvetica, sans-serif.
- font-weight: 700.
- font-style: italic.
- Sin descarga asociada.

ALGORITMO EXACTO
----------------
1. Normalizar title a Unicode NFC.
2. Convertirlo a minúsculas.
3. Eliminar espacios, signos y símbolos.
4. Conservar letras, tildes, ñ y números.

   "Dos días después del huracán"
   -> "dosdíasdespuésdelhuracán"

5. Contar los caracteres Unicode resultantes.
6. Dividirlos secuencialmente en tres grupos tan iguales como sea posible.
7. Si sobran caracteres, asignar uno primero a la línea 1 y después a la línea 2.
8. No buscar cortes semánticos y no modificar el orden del titular.
9. Dibujar las tres líneas en un bloque de 960 x aproximadamente 840 unidades,
   centrado dentro del lienzo 1200 x 1200.
10. Usar estas posiciones:

    línea 1: x=120, y=370
    línea 2: x=120, y=650
    línea 3: x=120, y=930

11. Cada línea utiliza textLength=960 y lengthAdjust="spacingAndGlyphs" para
    formar un bloque tipográfico cuadrado y compacto.
12. Usar font-size=300, Arial Bold Italic.
13. Escapar &, < y > como XML.
14. Minificar el resultado a una sola línea.

PLANTILLA SVG
-------------
<svg xmlns="http://www.w3.org/2000/svg" width="1200" height="1200" viewBox="0 0 1200 1200"><rect width="1200" height="1200" fill="#f7f7f5"/><g fill="#2d2d2b" font-family="Arial,Helvetica,sans-serif" font-size="300" font-weight="700" font-style="italic"><text x="120" y="370" textLength="960" lengthAdjust="spacingAndGlyphs">LINEA1</text><text x="120" y="650" textLength="960" lengthAdjust="spacingAndGlyphs">LINEA2</text><text x="120" y="930" textLength="960" lengthAdjust="spacingAndGlyphs">LINEA3</text></g></svg>

PROHIBIDO EN EL SVG
-------------------
- <metadata>, comentarios, RDF, EXIF o datos del programa creador.
- Fuentes incrustadas, base64, imágenes raster o filtros.
- Paths que conviertan las letras en curvas.
- JavaScript, enlaces, animaciones o elementos externos.
- Identificadores, clases o grupos innecesarios.
- XML generado por Illustrator/Inkscape con cabeceras y namespaces extra.

PESO
----
El objetivo normal es inferior a 1 KiB por SVG. La dimensión 1200 x 1200 no
incrementa significativamente el peso porque el archivo contiene texto y formas,
no 1.440.000 píxeles almacenados.

INTEGRACIÓN
-----------
El compilador incluido ya implementa TITLE GRID V1. El plugin del rack debe usar
el mismo algoritmo para que la previsualización local y la publicación sean
idénticas. Aunque el plugin no se ejecute, GitHub Actions regenerará el SVG desde
title al publicar.

La página HTML utiliza el SVG en:
- el bloque visual COMPARTIR / SVG / COPIAR LINK;
- og:image;
- twitter:image;
- NewsArticle.image;
- sitemap.xml como imagen asociada.

VALIDACIÓN AUTOMÁTICA
---------------------
La publicación falla deliberadamente si encuentra:
- JSON inválido;
- id o slug duplicados;
- slug con mayúsculas, espacios o caracteres no permitidos;
- fechas sin zona horaria;
- noticia sin title o content;
- URL de fuente inválida.

Esto impide que una ejecución automática rompa toda la web.
